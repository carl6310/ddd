# UI 产品巡检汇报：效果图对齐与显示完整性

日期：2026-05-03  
范围：项目、工作台、资料卡、提纲、正文、发布 6 个主界面  
依据：桌面 1512x982、平板 1024x768、手机 390x844 的真实页面截图与 DOM 溢出检测  
证据目录：`output/playwright/product-audit-2026-05-03/`

## 总体结论

当前界面已经具备左侧导航、顶部操作区、主内容区、右侧状态区的基本产品骨架，但还没有达到效果图里的“完整工作台”质感。主要差距不在颜色，而在布局密度、信息层级、响应式收纳、真实内容长度控制。

最需要优先修的是两个问题：

1. 窄屏下顶部操作按钮真实横向溢出，正文和发布页面会被按钮撑宽。
2. 多个页面把桌面三栏直接压到平板和手机，没有重新设计导航、操作区和右侧信息区的收纳方式。

桌面端没有检测到全局横向溢出，但存在大量首屏信息被截断、右侧栏内容下沉、长文本卡片难扫读的问题。手机端问题更严重，左侧导航会落到页面内容后面，发布和正文顶部按钮明显显示不完整。

## P0：必须先修的显示问题

### 1. 手机端顶部操作区横向溢出

影响页面：正文、发布  
证据截图：

- `mobileTop-正文.png`
- `mobileTop-发布.png`

现象：

- 正文页顶部的“保存 / 专注模式 / 版本历史 / 继续写作”横向排开，右侧按钮被裁掉。
- 发布页顶部的“导出 Markdown / 生成 DOCX / 复制公众号版 / 确认发布”直接撑出 390px 屏幕。
- 自动检测结果显示：手机正文 body 宽度 444px，手机发布 body 宽度 570px，已经超过 390px 视口。

产品判断：

这是明确的显示错误，不是审美问题。手机端不能照搬桌面按钮组，必须变成“主按钮 + 更多操作”或底部操作条。

建议：

- 手机端只保留一个主操作按钮。
- 次级操作进入更多菜单。
- 不可用按钮不要占据主操作区，放入更多菜单或隐藏。

### 2. 手机端导航位置不成立

影响页面：全部主页面  
证据截图：

- `mobile-正文.png`
- `mobile-发布.png`

现象：

- 桌面左侧导航在手机端没有转成可用的顶部/底部导航，而是出现在页面内容后面。
- 用户必须滑很久才能看到“项目、工作台、资料卡、提纲、正文、发布”入口。

产品判断：

这会让手机端几乎不可导航。即使项目主场景偏桌面，也不应该出现导航落在内容尾部的状态。

建议：

- 手机端改为顶部横向阶段切换，或底部固定导航。
- 侧边栏的项目进度卡在手机端应变成页面顶部摘要卡或折叠入口。
- 手机端不要保留桌面 sidebar 的完整结构。

### 3. 平板端顶部标题区被严重挤压

影响页面：发布最明显，正文、提纲、资料卡也存在  
证据截图：

- `tabletTop-发布.png`

现象：

- 平板发布页顶部标题区域被挤成很窄一列，标题显示成“发布...”。
- 保存信息被拆成多行，和右侧按钮组争抢空间。
- 顶部栏视觉重心混乱，不像效果图中的清晰 header。

产品判断：

1024px 是非常常见的窗口宽度，不能按“移动端例外”处理。平板应有独立断点，而不是桌面布局硬压缩。

建议：

- 1024px 左右隐藏品牌长标题或压缩为图标。
- 页面标题区单独占一行，操作区单独一行。
- 长操作按钮收纳为“主按钮 + 更多”。

## P1：桌面端主要布局差距

### 4. 左侧导航和效果图差距大

影响页面：全部桌面页面  
证据截图：

- `desktop-项目.png`
- `desktop-工作台.png`

现象：

- 当前左侧导航整体偏灰，品牌、图标、文字的对比度不足。
- 效果图里左侧导航是清晰的产品锚点：品牌标题、图标、当前态、进度卡都更稳定。
- 当前图标有些像占位符，不能形成专业工具感。

产品判断：

左侧导航是整个工作台的“骨架”。如果它偏弱，页面会显得像临时后台，而不是完整产品。

建议：

- 提高品牌文字与导航文字对比度。
- 统一图标尺寸和风格。
- 当前态保持蓝色高亮，但减少灰底面积的糊感。
- 底部进度卡保留，但应和效果图一致使用 32% 项目总进度，而不是当前项目 80% 阶段进度。

### 5. 工作台阶段条不够像效果图

影响页面：工作台  
证据截图：

- `desktop-工作台.png`

现象：

- 10 个阶段按钮在桌面端虽然没有溢出，但 ThinkCard、StyleCore 被截断成 `Thin...`、`Style...`。
- 效果图中阶段条更像流程导航，数字、阶段名、分隔关系清楚。
- 当前阶段条挤在一个卡片内，视觉上更像一排胶囊按钮。

产品判断：

工作台页的核心是流程感。阶段条如果读不清，会削弱“从选题到发布”的产品主线。

建议：

- 桌面端给阶段条固定横向滚动或更合理的分布宽度。
- 英文产品模块名不要省略到不可读。
- 发布阶段弱态可以保留，但不要和已完成阶段同等占位。

### 6. 资料卡页右侧详情栏过长，首屏关系不清

影响页面：资料卡  
证据截图：

- `desktop-资料卡.png`

现象：

- 左侧资料卡列表与右侧详情都在首屏出现，但右侧“高亮引用”内容过长，向下延伸明显。
- 真实资料只有 1 张，大量补位卡虽然提升了密度，但容易被误认为真实资料。
- 效果图右侧详情更像精选信息摘要，当前更像把长文本直接塞进详情面板。

产品判断：

资料卡页应该帮助用户快速判断资料是否可用，而不是让右侧详情变成另一个长文阅读区。

建议：

- 右侧详情首屏只展示摘要、引用、关联章节和证据强度。
- 长引用默认折叠，点击展开。
- 补位卡必须明确是“待补资料入口”，不要与真实资料卡同权重。

### 7. 正文页信息过载，写作焦点不够稳定

影响页面：正文  
证据截图：

- `desktop-正文.png`

现象：

- 顶部指标、左侧大纲、中间成文版、右侧建议三块同时出现，信息量过大。
- 左侧大纲标题和摘要大量截断。
- 中间文章阅读区可用，但右侧栏和左侧栏会抢注意力。
- 效果图中的正文页更像“编辑器 + 大纲 + AI 建议”，当前更像多个状态面板堆在一起。

产品判断：

正文页的首要任务是写作和编辑，状态诊断应该服务写作，而不是和正文同等竞争。

建议：

- 默认强化中间编辑区宽度。
- 左侧大纲只保留标题和状态，摘要改为 hover/展开。
- 右侧建议压缩为卡片列表，优先展示可执行建议。
- 顶部指标可以减少到 3 个核心项。

### 8. 提纲页右侧指标栏过窄，纵向卡片不自然

影响页面：提纲  
证据截图：

- `desktop-提纲.png`

现象：

- 右侧“证据 / 连续性 / 复查项”被挤成窄竖条。
- 主区域大量段落字段卡片向下延伸，首屏看不到完整章节编辑闭环。
- 效果图中的提纲页是结构树 + 当前章节详情 + 右侧检查，当前右侧检查缺少可读宽度。

产品判断：

提纲页应该让用户一眼知道当前章节的问题和改法。现在右侧指标太窄，反而像装饰。

建议：

- 右侧指标统一成横向小卡，不要竖排窄卡。
- 当前章节详情优先展示“主判断、支撑论据、缺失项、下一步”。
- 长字段区域折叠，避免首屏全是输入卡。

### 9. 发布页和效果图差距最大

影响页面：发布  
证据截图：

- `desktop-发布.png`

现象：

- 效果图发布页是三栏：发布前检查、文章预览、质量评估和待办。
- 当前发布页顶部是统计条，左侧检查清单较简，右侧没有完整质量评分区。
- 文章预览区域过长，右侧评估和导出版本没有形成强产品感。
- 多个按钮处于禁用状态但仍占据主操作位置。

产品判断：

发布页是最终交付页，应该给用户“可以发布 / 还差什么 / 导出什么版本”的确定感。当前更像草稿预览页。

建议：

- 恢复效果图中的质量评分、HKR 摘要、资料绑定完整度、待处理事项四块。
- 中间文章预览高度固定，内部滚动。
- 导出版本独立成列表，不放在顶部按钮里挤。
- 禁用操作要给原因，不能只是灰掉。

## P2：设计一致性问题

### 10. 当前进度含义混乱

现象：

- 左侧进度卡显示 80%，但效果图显示当前项目进度 32%。
- 项目页、工作台、发布页都在展示不同类型的进度，但没有清楚区分“项目总进度”和“当前阶段完成度”。

建议：

- 左侧固定为项目总进度。
- 页面内部显示当前阶段指标。
- 文案明确区分“项目进度 / 发布清单 / 写作健康度”。

### 11. 图像和占位素材缺少真实感

现象：

- 当前项目卡、资料卡使用大量斜纹占位图和图表占位。
- 效果图里很多卡片使用真实城市图、文件封面、机构 Logo，产品可信度更高。

建议：

- 至少为项目封面、资料卡封面、正文预览建立稳定的真实图片/封面规则。
- 没有真实图时，使用更克制的文件封面，不要大面积斜纹。

### 12. 禁用按钮太多，降低完成感

现象：

- 发布页检测到 10 个可见禁用按钮。
- 提纲、正文页也有多个禁用操作。

建议：

- 主流程里减少灰色不可点击按钮。
- 禁用按钮需要说明原因，或收进二级菜单。
- 用户当前不能做的事，不要放在最显眼的主操作区。

## 建议修复顺序

1. 先修响应式基础：手机和平板顶部操作区、导航收纳、横向溢出。
2. 再修发布页：按效果图重建发布检查、文章预览、质量评估、导出版本。
3. 修正文页：强化中间写作区，压缩左右辅助区。
4. 修资料卡页：真实资料和补位卡降噪，右侧详情折叠长引用。
5. 修提纲页：右侧指标栏和章节详情层级。
6. 最后统一左侧导航、阶段条、进度语义和占位素材。

## 验收标准

- 390px、1024px、1512px 三个宽度均无横向溢出。
- 顶部操作区在手机和平板不裁切文字。
- 左侧导航在手机端不出现在页面内容尾部。
- 发布页首屏能同时看到发布检查、文章预览、质量评估、待办摘要。
- 正文页首屏以文章编辑区为视觉焦点。
- 长文本卡片默认不把布局撑开，必须有截断、折叠或内部滚动策略。
