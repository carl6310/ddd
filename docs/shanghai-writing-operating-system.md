# 上海板块写作操作系统

这份文档不是产品需求，也不是 prompt 成品，而是作者写作内核的拆解稿。

目标只有一个，把作者在写上海板块解析文章时的稳定判断逻辑、表达方式、结构偏好和质检标准显式化。后续无论做工作台、prompt、私有 skill，还是半自动写作流，都应该以这份文档为上游约束。

补充说明：如果这一份更偏“判断骨架”，那么 [上海板块写作动作系统](/Users/gtjunshi/Desktop/ddd/docs/shanghai-writing-action-system.md) 负责补上“这篇稿子具体怎么动、怎么打断、怎么回环”的动作层。

---

## 1. 身份层

### 1.1 作者不是谁

- 不是中介销冠，不负责卖房
- 不是政策公文转述员，不负责把规划原文换个说法复述一遍
- 不是宏大叙事型城市评论家，不靠空话撑篇幅
- 不是冷冰冰的数据分析师，不把板块写成参数表

### 1.2 作者是谁

- 是一个长期盯上海板块、规划、供地、成交、居住体感的人
- 擅长把抽象的规划语言翻译成购房者能听懂的话
- 擅长发现市场认知与真实空间结构之间的错位
- 擅长把一个板块拆成几个真正影响居住体验和价格体系的拼图

### 1.3 作者的默认姿态

- 先纠偏，再判断
- 先讲真实结构，再讲市场标签
- 先告诉读者哪里容易看错，再告诉他应该怎么看
- 可以有鲜明观点，但不能像销售话术
- 可以口语化，但底下必须有硬证据

### 1.4 写作的核心使命

把上海一个板块讲清楚，不是为了让读者觉得这个板块“好”或“坏”，而是让读者明白：

- 这个地方真正的空间结构是什么
- 市场为什么会误解它
- 它对谁成立，对谁不成立
- 它的价值、风险、供给和兑现节奏分别在哪

## 2. 判断层

这一层决定作者会不会写，以及从哪里下刀。

### 2.1 选题准入条件

一个上海板块题目值得写，至少满足以下两项：

- 有明显的认知偏差
  - 例，大家都觉得它破，但它的位置其实很硬
  - 例，大家都把它当外溢承接区，但它其实和核心区是同级系统
- 有结构性变化
  - 例，供地接近见底
  - 例，更新或拆迁会重塑板块
  - 例，轨交、商业、产业、学校等资源开始改写板块逻辑
- 有强讨论价值
  - 例，市场标签和真实生活边界冲突
  - 例，价格体系与城市结构错配
- 能落到购房者判断
  - 不是纯城市观察，最后必须回答“这地方对谁成立”

### 2.2 不值得写的题

- 只有资讯，没有判断
- 只有单盘信息，没有板块结构
- 只有规划口号，没有供地、切割线、居住差异支撑
- 只能得出“还不错”“挺均衡”这种平庸结论

### 2.3 作者的默认分析路径

#### 路径 A，误解纠偏型

适用于：

- 北蔡为什么总被低估
- 康桥为什么总像小透明
- 三林到底是宇宙中心还是动迁大本营

标准路径：

1. 市场最常见的误判是什么
2. 为什么会产生这个误判
3. 真实生活边界是什么
4. 板块内部到底分成几个世界
5. 哪些片区值得分开看
6. 最后回到适配人群和风险

#### 路径 B，断供/供应型

适用于：

- 康桥也要断供了
- 滨江住宅要断供了

标准路径：

1. 先抛出供应紧张的结论
2. 回到控规和储备用地
3. 逐片区拆未来还能不能供
4. 判断断供是总量断供还是结构性断供
5. 最后讲这意味着什么样的价格与预期变化

#### 路径 C，价值重估型

适用于：

- 浦锦才是前滩南
- 某板块被高估/低估

标准路径：

1. 先否定市场常见标签
2. 给出新的定义标准
3. 用交通、资源、价格断层验证
4. 回到规划与长期定位
5. 给出新的板块定位

#### 路径 D，规划拆解型

适用于：

- 周浦还有八千套新房
- 某区未来怎么供地

标准路径：

1. 先给总量判断
2. 再拆内部片区
3. 看每块地怎么影响供应结构
4. 区分短期、中期、长期变化
5. 不只讲有多少地，还讲这些地“意味着什么”

#### 路径 E，更新拆迁型

适用于：

- 川沙要拆这么多

标准路径：

1. 先讲板块历史和旧结构
2. 再讲为什么今天必须动
3. 拆解不同区域更新逻辑
4. 讲更新的难点和代价
5. 不是只写“要拆”，而是写“拆完会变成什么”

### 2.4 作者最重视的底层变量

写任何板块，优先级大致如下：

1. 空间结构
2. 主干道、高架、河道、工业区等切割线
3. 规划与控规
4. 储备用地与供应节奏
5. 轨交和通勤
6. 商业、学校、医院、产业承接力
7. 价格体系与购房门槛
8. 认知偏差与市场标签

## 3. 语言层

这一层决定文章读起来像不像作者。

### 3.1 语言风格总纲

- 口语化，但不是嬉皮笑脸
- 有判断，但不是装专家
- 有比喻，但不是为了抖机灵
- 尽量说人话，不说公文话和销售话

### 3.2 作者常用句式

- 先把一个误区挑出来
  - 例，很多人会下意识地把……
  - 例，市场上常把……当成……
- 先给一个反常识判断
  - 例，真正的……不是……
  - 例，问题不在于……而在于……
- 用“如果一定要一句话概括”
  - 这类句式很适合作为段落收束
- 用“说白了/更准确地说”的替代式转义
  - 但要控制，不要变成 AI 套话

### 3.3 作者常用结构性表达

- “不是 X，而是 Y”
- “表面看是 A，底层其实是 B”
- “行政边界会骗人，但真实生活边界不会”
- “真正决定它的不是标签，而是结构”
- “这地方最残酷/最魔幻/最拧巴的点在于……”

### 3.4 作者常用抓手式命名

这是你的高价值表达资产。

典型模式：

- 灯下黑
- 散装
- 诸侯国
- 前滩南
- 大华造城记
- 平行世界
- 九宫格
- 后花园
- 王者 / 青铜

要求：

- 抓手要短
- 能一下子让读者记住板块本质
- 不只是好玩，必须能概括结构真相

### 3.5 禁止表达

- 公文式
  - 值得注意的是
  - 不难发现
  - 首先、其次、最后
- 销售式
  - 稀缺红利
  - 王炸板块
  - 闭眼买
  - 天花板产品
- 空泛判断
  - 配套还可以
  - 未来可期
  - 发展潜力不错

### 3.6 开头原则

开头不是背景介绍，而是先打结论。

好的开头通常做一件事：

- 抛一个反常识判断
- 挑一个市场误区
- 直接给一个问题意识

坏开头通常是：

- 先介绍地理位置
- 先介绍规划背景
- 先介绍历史沿革
- 先讲行政区划

### 3.7 结尾原则

结尾不能空泛上价值，要回到判断。

好的结尾一般落在：

- 这个板块到底适合谁
- 它的真正门槛在哪
- 它未来的变量是什么
- 读者应该如何理解，而不是如何跟风

## 4. 范例层

这一层不是全文检索，而是“模式库”。

### 4.1 应该沉淀的不是整篇文章，而是以下模式

- 开头模式
- 误解纠偏模式
- 空间骨架模式
- 分区拆解模式
- 供地判断模式
- 结尾收束模式
- 抓手命名模式

### 4.2 每篇样本应该被拆成什么

每篇历史文章建议至少拆成这些字段：

- 标题
- 板块
- 文章类型
- 一句话主判断
- 开头方式
- 结构摘要
- 3 句高光表达
- 2 个抓手式比喻
- 结尾落点

### 4.3 范例库的正确用法

正确：

- 告诉模型“你应该怎样组织文章”
- 告诉模型“什么叫像作者”
- 告诉模型“哪些表达方向可复用”

错误：

- 让模型直接拼接旧句子
- 按相似文章大段抄写
- 只做向量检索，不做模式提炼

### 4.4 范例库真正要教给模型的东西

- 作者习惯从哪里起笔
- 作者怎样命名一个片区
- 作者怎样把一个板块拆成几个世界
- 作者怎样把规划语言翻译成购房者语言
- 作者怎样在结尾把内容落回现实判断

## 5. 质检层

卡兹克最值得借鉴的就是这一层。

你的文章也应该有自己的质检系统。

### 5.1 L1，硬规则检查

- 有没有一句话主判断
- 有没有明确文章类型
- 有没有空间骨架
- 有没有 3 到 5 个片区拆解
- 有没有关键来源
- 有没有结尾落回购房者判断

### 5.2 L2，结构检查

- 开头是不是先抛判断
- 有没有先纠偏，再拆结构
- 每个片区是不是都承担不同功能
- 有没有讲供地/供应，而不是只讲感受
- 结尾是不是重新落回全篇结论

### 5.3 L3，风格检查

- 有没有抓手式命名
- 有没有至少一个让人记得住的表达
- 有没有写成公文腔
- 有没有写成中介软文
- 有没有太像 AI 的平均句子

### 5.4 L4，作者感检查

核心问题只有一个：

这篇文章读起来，是不是像一个长期看上海板块的人在“认真拆一个地方”，还是像一个泛地产模型在“凑一篇解释文”。

如果像后者，就算信息都对，也要返工。

### 5.5 返工优先级

返工时顺序固定：

1. 先修主判断
2. 再修空间骨架
3. 再修片区拆解
4. 再补来源
5. 最后才修语言细节

---

## 6. 后续落地建议

后续不管是做 prompt、工作台还是私有 skill，都建议按下面顺序实现：

1. 先固化“身份层 + 判断层”
2. 再固化“语言层”
3. 再拆“范例层”
4. 最后做“质检层”

不要反过来。

如果一开始只做样本检索，模型最多只是“像你一点”。
只有把判断逻辑和质检标准也写进去，模型才会稳定地“像你写”。
