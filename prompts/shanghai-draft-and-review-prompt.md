# 上海板块正文与质检提示词

> 用途  
> 用于根据提纲和资料卡生成正文，或对已有正文做“像不像作者”的质检。  
> 推荐分两次调用：  
> 1. 正文生成  
> 2. 质检与返工建议

---

## A. 正文生成版

请基于以下输入，生成一篇“上海板块解析文章”。

### 输出要求

你必须输出 JSON，不要输出额外说明。

```json
{
  "analysisMarkdown": "强调证据和逻辑的 Markdown 稿件",
  "narrativeMarkdown": "强调可读性和文章感的 Markdown 稿件"
}
```

### 硬性要求

1. 必须有一句话主判断
2. 必须先纠偏，再拆结构
3. 必须解释空间骨架，而不是只报信息
4. 必须拆成 3 到 5 个片区
5. 关键判断后必须挂资料卡引用，格式固定为 `[SC:sourceId]`
6. 结尾必须回到“这个板块对谁成立，对谁不成立”
7. 必须落一个具体人物 / 生活场景、一处文化升维、一句短促断裂句，并让结尾回扣开头
8. 绝对不能写成中介软文

### 风格要求

- 学习历史样本中的反常识开头、抓手式命名、空间拆解和口语节奏
- 可以模仿表达方向，但禁止复用历史样本原句
- 不要写成公文腔
- 不要写成销售腔
- 不要写成百科介绍

#### 选题

{{topic}}

#### 文章类型

{{article_type}}

#### 一句话主判断

{{thesis}}

#### 核心问题

{{core_question}}

#### 段落级提纲

{{outline_json}}

#### 板块建模

{{sector_model_json}}

#### 资料卡

{{source_cards_summary}}

#### 历史样本模式

{{style_reference}}

---

## B. 质检版

请对下面这篇上海板块解析文章做“作者感质检”。

### 输出要求

你必须输出 JSON，不要输出额外说明。

```json
{
  "overallVerdict": "整体判断",
  "completionScore": 0,
  "checks": [
    {
      "key": "check-id",
      "title": "检查项",
      "status": "pass|warn|fail",
      "detail": "具体说明",
      "evidenceIds": ["source card id"]
    }
  ],
  "revisionSuggestions": ["建议"],
  "preservedPatterns": ["保留住的个人风格"],
  "missingPatterns": ["缺失的个人风格"]
}
```

### 质检重点

你必须判断下面这些问题：

1. 开头是不是先抛判断，而不是先讲背景
2. 有没有讲清板块内部是怎么被切开的
3. 有没有拆成 3 到 5 个真正不同的片区
4. 有没有供地、规划、更新层面的硬支撑
5. 有没有写成中介软文
6. 有没有 AI 套话或公文腔
7. 读起来像不像一个长期看上海板块的人在认真拆一个地方
8. 有没有新的具体观察，而不是旧结构复写
9. 有没有人物体温、回环和现实代价

### 返工优先级

如果不合格，建议必须按这个顺序给：

1. 先修主判断
2. 再修空间骨架
3. 再修片区拆解
4. 再补来源
5. 最后才修语言细节
