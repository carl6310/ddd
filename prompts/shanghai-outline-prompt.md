# 上海板块提纲提示词

> 用途  
> 输入一个选题、研究资料和历史样本模式，输出段落级提纲。  
> 目标不是“列目录”，而是生成一份能直接交给正文生成器或作者本人继续写的结构稿。

---

请基于以下输入，生成一份“上海板块解析文章”的段落级提纲。

## 你的任务

你不是简单列大纲，你要产出一份“段落任务书”。

每一段都必须说明：

- 这一段要证明什么
- 用哪些证据
- 节奏和情绪是什么
- 这一段执行的写作动作是什么
- 这一段怎么把读者带到下一段
- 这一段结束时读者应该得到什么判断

## 输出要求

你必须输出 JSON，不要输出额外说明。

```json
{
  "hook": "开头一句",
  "sections": [
    {
      "id": "section-1",
      "heading": "段落标题",
      "purpose": "这一段要证明什么",
      "evidenceIds": ["资料卡ID"],
      "tone": "节奏/情绪说明",
      "move": "这一段执行的写作动作",
      "bridge": "这一段如何承接到下一段",
      "keyPoints": ["要覆盖的点"],
      "expectedTakeaway": "读者看完这一段会得到什么"
    }
  ],
  "closing": "结尾要落到哪里"
}
```

## 结构硬约束

1. 必须先有一个反常识或纠偏开头
2. 必须先解释“为什么大家会看错”
3. 必须单独有一段讲空间骨架
4. 必须拆成 3 到 5 个片区
5. 必须有一段讲供地/供应/更新对未来走势的影响
6. 结尾必须落到购房者判断，不允许空泛收尾

## 风格要求

- 提纲应该像作者本人会写的文章，而不是咨询报告目录
- 每个片区标题尽量有抓手式命名
- 不要用“第一部分 / 第二部分 / 第三部分”这种平铺标题
- 不要只写抽象标题，必须写清楚这一段的工作
- 至少有一段要落具体人物或生活场景
- 至少有一段要负责文化升维
- 结尾段必须明确写出回环动作

## 最后提醒

- 你是在给“上海板块解析”写提纲，不是在给知识科普文写提纲
- 读者最关心的是“哪里容易看错、结构怎么拆、对谁成立”
- 提纲必须体现作者的判断力，而不是只体现信息量
